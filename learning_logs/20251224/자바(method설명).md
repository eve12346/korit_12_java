# Method
## 정의
JAVA에서는 함수(function)의 개념과 동일함. JavaScriopt /
Python에서는 function / method 개념이 분리되어 있으나
Java에서는 모든 함수가 클래스에 종속되어 있기 때문

function vs. method
    그렇다면 용어를 왜 분리하는가?
    `기본적으로 클래스 내부에 함수를 정의하게 되면 이를
    '메서드'라고 합니다. java는 모든 함수들이 특정 클래스내부에 있습니다.
    (저희가 .java 파일을 생성할 때 클래스를 만든다고 하죠, 그리고 그 내부에 method들을 정의하게 되기 때문에 클래스 내부에 함수가 있다고 해석할 수 있습니다.)
    그래서 다른 언어를 배우게 된다면 '함수의 호출 방식'과 '메서드의 호출방식' 상에서의 차이가 존재하지만, 현재 저희 수업을 기준으로 했을 때는
    전부 다 method에 해당하고, 제거 Java 수업 중에 함수 어쩌고 하면 그냥 method라고
    알아들으시면 됨니다(타 언어 수업 때는 좀 더 엄밀히 구분할 예정.)

## 용어 정리
    1. method 정의(define)  : 사용자가 메서드를 새로 만드는 것을 의미
    2. method 호출(call)    : 만들어진( 혹은 미리 만들어진) 매서드를 실행시키는
    것을 의미
    3. 매개변수(perameter) : 특정 메서드에 필수적으로 요구되어지는 변수 - 정의 시에 요구됨
    4. 출력값 / 결과값 / 리턴값(return) : 메서드의 호출 결과 생성되는 자료(없을 수도 있음/output에 해당)
    5. 인수 / 인자(argument) : 사용자가 메서드에 전달하는 입력(input) - 호출 시에 요구됨

## Java 상에서의 method

예를 들어, '사진을 찍는다' 라는 행위에 대해서 설명하자면
1. 주머니에서 폰을 꺼내고
2. 잠을 화면을 풀고( 그런데 폰마다 Face ID인 경우가 있고 패턴인 경우도 있고 etc)
3. 카메라를 켜고
4. 사진을 찍고자 하는 대상에 폰을 조준하고
5. 셔터를 누른다
라는 해당 과정이 존재합니다. 그런데 컴퓨터는 생각보다 너무 멍청하기 때문에 '사진을 찍는다' 라는 명령을 내리는 것으로는 충분하지 않고, 1-5까지의 명령어를 하나하나 '순서대로' 내려야 비로소 '사진을 찍는 행위'가 완료되게 됩니다

그런데 사진 한 장 찍고 싶을 때마다 1-5를 매번 코드를 입력한다고 하면 너무 귀찮을겁니다. 그래서 1-5에 해당하는 명령들을 순서대로 묶어놓는 행위를 미리 해두는 방법을 사용하는데 이를 'method 정의'라고 하게 됩니다.

그리고 실제로 컴퓨터에게 '사진을 찍는다' 라는 명령을 내리게 되는 것을 'method 호출'이라고 표현할 수 있겠습니다.

여러분의 폰이 아이폰인지 안드로이드폰인지에 따라서 잠금화면을 푸는 방법도 다르고, 사진을 찍는 방식도 다를겁니다. 그렇다면 특정 폰을 가지고 있음을 컴퓨터에게 알릴 필요가 있을건데, 이를 argument라고 할 수 있겠습니다. 그리고 사진을 찍으면 사진 결과물이 남을텐데, 이를 return이라고 말할 수 있습니다.

## method에서의 중대 원칙
- 하나의 메서드에는 하나의 기능만 있을 것.

## method overloading
- 매개변수의 유무 및 자료형의 차이, 개수의 차이를 나눠서 동일한 메서드 명을 지닌 메서드'들'을 정의할 수 있다.

- 즉, 메서드 명은 동일하고, 매개변수에 들어가는 자료형, 개수를 달리하게 되면 서로 다른 메서드로 인지하게 됩니다.

- 예를 들어서 숫자끼리 더하는 메서드가 있다고 가정하겠습니다. 걔를 add()라고 정했다고 칩시다.
```java        
        public int add(int a, int b) {
            return a + b;
        }

        근데 문자열끼리 더하는 게 필요해서 add는 못 쓰니까 connect를 씁시다.
        public String connect(String a, String b) {
            return a + b;
        }

        근데 뒤집어서 연결해야 할 때가 있어서
        public String connect(String a, String b) {
            return b + a;
        }

        근데 정수 더하기 실수 해야해서
        public double plus(int a, double b) {
            return a + b;
        }
```
이런 부분이 문제가 됩니다.

- method overloading의 장점 :
           동일한 기능을 하는 메서드를 정의할 때 메서드 명이 필요한데,
           예를 들어 String 자료형끼리 연결해주는 메서드와
           int끼리의 합 연산을 하는 메서드를 나눈다고 가정했을 때,
           add,
           plus,
           connect 등을 정의했다면,
           다음 합 연산을 하는(예를 들어 double 자료형을 지니는) 메서드 명을 정하려고
           하면 번거롭습니다.
           이러한 부분을
```java           
           public void add(String example1, String example2) {
               로직
           }
           public void add(int example1, int example2) {
               로직
           }
           public void add(double example1, double example2) {
               로직
           }
           과 같이 자료형을 달리 하는 '동일한 이름을 가지는 메서드들'을 정의하거나,
          public void add(double example1, double example2, double example3) {
               로직
           }
```
와 같은 식으로 매개변수의 개수의 차이를 두더라도 '동일한 기능을 하는 서로 다른 메서드'들로 인식하게끔 하는 방식입니다.

그러면 메서드 명들도 동일하고, 걔네가 뭐 매개변수 유형이가 개수가 차이가
있겠지만 기능도 같으니까 머리가 안아프다는 장점이 있다고 할 수 있습니다.
